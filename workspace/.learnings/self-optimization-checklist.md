# 系统自优化检查清单

> 每次心跳/任务完成后执行，2026-06-29 创建

## 心跳自检（约每60分钟）

1. **KBI扫描**：读 `.learnings/kbi-tracker.json`
   - 未应用的ROI≥80条目 → 自动应用（低风险）或生成方案（中高风险）
   - 已应用>30天零触发 → 标记待废弃

2. **LEARNINGS提取**：读最近2天的 `memory/YYYY-MM-DD.md`
   - 提取用户纠正/新规则/新发现 → 写入 LEARNINGS.md
   - 同类≥2次 → 升级为规则写入 MEMORY.md

3. **ERRORS扫描**：读 `.learnings/ERRORS.md`
   - 同类错误≥2次 → 自动生成修复脚本
   - 3次 → 固化到系统规则

4. **质检统计**：如果期间有小说产出
   - 跑 qc-novel.js 全量扫描
   - 绿灯数/红灯数 → 写入每日日志

5. **主动优化建议**：
   - 发现可优化模式（任务重复、等待浪费、命名不一致）→ 自动提出改进方案
   - 不等用户要求，下次交互主动汇报"我发现可以优化的几点"

## 任务完成自检

- 子Agent返回的 learnings/blockers/suggestions → 提取写入 KBI
- 本任务中踩的坑 → 写入 ERRORS.md
- 本任务中发现的技巧 → 写入 LEARNINGS.md
