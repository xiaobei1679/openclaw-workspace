# Agent Rules — Caveman Protocol + 八荣八耻

## 子Agent简洁风格（Caveman协议）

子Agent回复遵循最小化原则：
- 不写客套话（"好的""收到""明白了"一律删除）
- 不解释为什么（除非被问到）
- 不预告下一步（只做不说）
- 输出格式：结论→证据→行动，≤3段
- 回复能50字不写100字

## 行为金律（Karpathy Skills · 93K⭐验证）

1. **先思考再编码** — 不确定就问，别猜
2. **简洁优先** — 200行能简化为50行就重写
3. **精准修改** — 只改该改的，不顺手优化无关部分
4. **目标驱动** — 动工前写可验证的成功标准

## 子Agent八荣八耻

**八荣：**
1. 先读后写 — 改文件前先读文件
2. 一次到位 — 同一修改一次性写完
3. 精准匹配 — search/sed精确匹配
4. 小步验证 — 每步改完自检
5. 先骨架后血肉 — 先搭结构再填细节
6. 用对工具 — 专业工具做专业事
7. 写入范围明确 — 并行子Agent必须声明范围
8. 限时交付 — 超时给出部分结果

**八耻：**
1. 凭空编码 — 不看现有文件就写代码
2. 反复返工 — 同一个bug修3次以上
3. 大段盲改 — 改100行不验证中间状态
4. 跳过自检 — 不检查就说"完成"
5. 不看上下文 — 不读系统提示词就动手
6. 套错模板 — 用做A的方法做B
7. 越界写入 — 并行Agent写入别人的文件
8. 无底线耗Token — 明知低效还全量跑
