# KI-20260629-012 | OpenClaw Token优化与记忆系统最佳实践

- **来源**：CSDN社区多篇文章综合，2026-06-29
- **L1 ROI评分**：适用性(3)×落地难度(2)×影响力(5)×有效期(3)×紧迫度(3) = **90** ✅ 采纳

## 摘要

### 四大Token消耗源
- 系统提示词固定税（SOUL.md/MEMORY.md/AGENTS.md 每次注入）
- 会话历史滚雪球（长对话上下文线性增长）
- 工具输出垃圾堆积（执行结果24h不过期）
- 心跳静默消耗（每次HEARTBEAT_OK也吃token）

### 核心优化策略（10条已验证）

**1. 上下文修剪（Context Pruning）- 效果最显著**
- 默认TTL 1小时→缩至5分钟 + hardClearRatio 0.5
- 工具结果失效后立即从上下文剔除
- 配置：`"contextPruning": {"mode": "cache-ttl", "ttl": "5m", "softTrimRatio": 0.3, "hardClearRatio": 0.5}`

**2. 会话压缩（Session Compaction）**
- safeguard(保守)→adaptive(激进)模式
- 实测可降低90% token（摘要压缩）
- 配合滑动窗口（保留5-20轮对话）

**3. 提示词缓存（Prompt Caching）**
- 针对Claude系列模型：缓存读取=写入成本×1/10
- 心跳设为55分钟（略低于缓存失效1小时），保活缓存

**4. Memory分类管理**
- 长期记忆：MEMORY.md（只存偏好/规则/铁律，不存项目细节）← 我们刚做了这个
- 短期记忆：24h自动清理的临时数据
- 自动总结：每10轮对话生成摘要替代完整历史

**5. 选择性加载（动态分配比例）**
- 系统提示词10%、记忆20%、对话历史40%、工具结果20%、预留10%
- 大文件拆分为子文件按需read ← 我们刚做了这个

**6. SOUL.md/系统提示词瘦身**
- 去掉冗余指令，只留核心人设
- 明确要求"精简回复"（已在MEMORY.md规则4）

**7. 本地Embedding替代云端API**
- memory_search 用本地模型 → 成本归零
- 适用场景：知识库检索、记忆搜索

**8. 子Agent并发控制**
- 默认并发8→降至4
- 减少同时间塞进上下文的工具结果量

**9. 任务拆解+分步执行**
- 一个复杂任务拆成多个小任务
- 每次只带相关上下文，减少每次token消耗

**10. 模型分层使用**
- 简单任务→轻量模型（hy3-preview），复杂任务→全量模型
- 心跳/自检/简单查询用便宜模型

### 记忆系统架构（OpenClaw Markdown记忆模型）
- 持久化：workspace/*.md → 注入为system context
- 会话级：LCM压缩→会话摘要（sum_xxx）
- 关键设计原则：
  1. 长期记忆(规则/偏好/铁律) → MEMORY.md常驻
  2. 项目细节(命名/设定/Agent) → 独立文件按需read
  3. 会话历史→LCM自动压缩摘要
  4. 临时数据→24h自动清理
