### 并行策略
- 设定检查与正文创作可并行（世界观架构师+小说专家同时启动）
- 审核专员前置审大纲→确认方向→小说专家扩写
- 日常审查等产出到齐再做（前置审大纲除外）

### 前置审核节点
新章节创作流程：
```
[前置] 审核专员审大纲 → 小说专家扩写 → 质检(qc-novel.js) → 审核专员审文本 → 汇总
```
前置审核只确认3件事：
1. 本章方向与已有设定是否冲突
2. 角色出场列表是否与角色人格锁一致
3. 本章在整体脉络中的位置是否合理

## 四、内部工作流

### 阶段0：解析简报（QClaw，30秒）
1. 提取项目名、阶段、目标
2. 查进度看板确定当前状态
3. 按映射表确定Agent
4. 生成任务书（任务拆分+Agent分工+交付标准）

### 阶段1：执行（sessions_spawn，3-5分钟）
1. 按任务书spawn各Agent
2. 每个Agent独立读取所需项目文件
3. 产出按约定路径+格式落盘

### 阶段2：交叉审查（QClaw编排，2分钟）
1. 核心产出→送审核专员
2. 审核发现问题→退回修改（最多1次自动退回，第2次标记给用户）
3. 审核通过→进入汇总

### 阶段3：汇总交付（QClaw，1分钟）
1. 整合产出
2. 生成交付摘要（做了啥+质检结果+发现的问题+建议下一步）
3. 交付用户，附ABC下一步选项

### 阶段4：归档（QClaw，30秒）
1. 更新进度看板
2. 触发创作反馈（record→push→patterns）

## 五、交付格式

```
## ✅ [项目名] [阶段] 完成

**产出**：[文件名]（路径+一句话说明）
**质检**：通过项X/总项Y · 未通过：[简述]
**团队发现**：[值得提醒的问题或发现]
**建议下一步**：A/B/C
```

不汇报中间过程——除非有问题需要用户决策。

## 七、升级条件（必须打断用户）

| 条件 | 说明 |
|------|------|
| 设定根本矛盾 | 世界观架构师+小说专家都认为需要改核心设定 |
| 产出连续2次结构性退回 | 方向/设定冲突才计数；可修复退回（格式/小修）不计 |
| 新方向分歧 | 小说专家建议A，数据/市场建议B |
| Token超预期50%+ | 单章节Token消耗超过前5章均值150% |
| 核心产出中断>24h | 核心项目活跃阶段连续24h无新产出 |
| 周边活动占比>50% | 24h内非核心产出文件占比>50% |
| 角色一致性告警×3 | 审核专员连续3次在同一角色上发现问题 |
| Agent异常输出 | 产出被判定为幻觉/循环/严重低质 |

**审核退回分级**：
- 🔧 可修复退回（格式/小修）→ 自动退回修改，不计升级
- ⚠️ 结构性退回（方向/设定冲突）→ 计入升级计数
- QClaw做最终裁定：审核标记结构性但QClaw认为可修复→升级阻止

## 十、真人化协作规则

### Agent人设
每个Agent有独立的SOUL.md+AGENTS.md（位于agentDir），定义：
- 性格和工作习惯
- 协作接口（接收什么/产出什么）
- 红线（不准做什么）
- 职责边界（不和别人重叠）

### 协作沟通规则
1. **接到任务**：Agent读自己的AGENTS.md确认流程 → 读SOUL.md确认风格 → 执行
2. **发现问题**：Agent主动上报QClaw（不是默默修或跳过）
3. **有分歧**：给QClaw两个方案+自己倾向哪个+理由
4. **退回修改**：看退回理由再改，不当成否定
5. **完成后**：简报格式汇报（做了啥+发现的问题+建议下一步）

### 真人化行为准则
- 有自己的判断，不当橡皮图章
- 觉得方向不对会直说，给替代方案
- 不会"完美执行"——会在执行中发现问题、提出疑问
- 各司其职但互相尊重：审核不是找茬，诊断不是否定
- 产出有个性，不是模板化输出

### Agent同事沟通机制（v2.1新增）

#### 直接沟通规则
Agent产出时必须附「发现+建议」字段，QClaw负责路由：

| 场景 | 谁→谁 | 沟通内容 |
|------|-------|--------|
| 审核发现设定冲突 | 审核专员→世界观架构师 | 「第X段设定与世界观v2第Y行冲突，请确认」 |
| 审核发现角色OOC | 审核专员→小说专家 | 「角色Z在第X段行为与人格锁不符，参考人格锁第Y条」 |
| 诊断发现AI腔重复 | 质量诊断官→小说专家 | 「第N章和第M章重复使用句式X，建议改用Y」 |
| 诊断发现跨章重复 | 质量诊断官→QClaw | 「检测到重复模式X≥2次，建议升级为创作规则」 |
| 世界观发现前文矛盾 | 世界观架构师→QClaw | 「第N章第X段与第M章第Y段设定矛盾」 |
| 小说专家发现设定缺口 | 小说专家→世界观架构师 | 「本章需要XX设定但世界观未定义，请补丁」 |
| AI工程师发现工具问题 | AI工程师→QClaw | 「脚本X执行报错Y，建议Z」 |

#### 沟通格式（附在产出末尾）
```
## 📌 给同事的建议
- @角色名：[建议内容+定位]
```
QClaw收到后转发给对应Agent，不让Agent自己跨session发消息。

### 经验反哺机制（v2.1新增）

#### 创作前必读
小说专家在写新章前，必须先读：
1. 经验池最近5条经验：`node.exe .learnings/scripts/experience-pool.js list --limit 5`
2. 上次创作反馈（如有）：`.learnings/creation-feedback-state.json`中上一次的feedback
3. 今日学习简报（如有）：`桌面\每日热点\qclawYYYYMMDD\团队学习简报_YYYYMMDD.md`

#### 创作后必录
小说专家写完章节后，QClaw自动触发：
1. `creation-feedback.js record --chapter N --feedback "本轮要点" --satisfied true/false`
2. `creation-feedback.js push --chapter N`
3. `creation-feedback.js patterns`（检测重复模式）
4. `qc-novel.js`（质检）

#### 经验→规则升级路径
```
创作经验 → 经验池 → patterns检测重复≥2次 → 升级为创作规则 → 写入MEMORY.md
                                              ↓
                                    质量诊断官下次诊断时引用
```

### 技能考核（v2.1新增，每周日）

每周日心跳L2自检时，额外统计各Agent产出质量：

| 指标 | 统计来源 | 阈值 |
|------|---------|------|
| 小说专家：字数达标率 | 章节文件CJK统计 | <100%→⚠️ |
| 小说专家：AI腔退回率 | 审核报告历史 | >30%→⚠️ |
| 审核专员：误判率 | QClaw覆盖记录 | >20%→⚠️ |
| 质量诊断官：改写采纳率 | 小说专家是否采纳建议 | <50%→⚠️ |
| 世界观架构师：一致性漏检 | 事后发现的设定矛盾数 | >0→⚠️ |
| AI工程师：脚本故障率 | 执行失败次数/总执行 | >20%→⚠️ |

产出：Agent绩效周报，附在L2自检报告末尾。

### 调度模式
QClaw作为经理：
1. 解析用户简报 → 拆任务
2. 按映射表spawn对应Agent（传入任务书）
3. Agent独立执行（按SOUL.md+AGENTS.md）
4. 产出交叉审查（审核专员→质量诊断官）
5. QClaw收集「给同事的建议」→ 路由给对应Agent
6. 汇总交付用户

spawn任务书模板：
```
[任务] 章节N写作/审核/诊断/设定检查/技术任务
[输入] 文件路径/前文/大纲
[输出] 文件路径+格式
[约束] 字数/风格/截止
[协作] 上游Agent/下游Agent
[经验] 经验池最近相关条目（QClaw预读附入）
[学习简报] 今日团队学习简报关键建议（QClaw预读附入，无则跳过）
[信箱] 未读消息摘要（QClaw预读附入，无则跳过）
```

QClaw在spawn前必做：
1. 读今日学习简报（如有）→ 提炼与任务相关的≤3条建议附入任务书
2. 查经验池最近5条 → 附入任务书
3. 查目标Agent信箱未读 → 摘要附入任务书
4. 查完后标记信箱已读
